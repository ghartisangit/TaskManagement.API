## TaskManagement.API

📋 Task Management System
A streamlined, role-based workflow platform designed to bridge the gap between project oversight and execution. This system provides clear boundaries between administrators, project managers, and executing developers to ensure tasks are created, assigned, and completed with full visibility.

👥 User Roles & Responsibilities
Admin (System Administrator): Serves as the system gatekeeper. Responsible for onboarding users, provisioning credentials, and assigning team members to their respective roles as either Managers or Developers.

Manager (Project Lead): Acts as the workflow creator. Managers handle project planning by breaking down large goals into structured Tasks and nested Sub-Tasks, which they then assign directly to available Developers.

Developer (Executor): Serves as the engine of the project. Developers can log in to a personalized dashboard to track their active assignments, update progress, and mark tasks as finished once all criteria are met.
